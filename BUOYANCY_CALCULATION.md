# 浮力计算推导

## 1. 船体参数
- 质量：37,000,000 kg (3.7万吨)
- 尺寸：138m × 95m × 85m
- 吃水深度：11m
- 重力：37,000,000 × 9.82 = **363,340,000 N**

## 2. 浮力点数量计算

### 当前浮力点生成逻辑：
- `lengthSegments = 9` → 10个位置（0到9）
- 每个位置3个横向点：左(-hx)、中(0)、右(hx)
- 船中区域（zRatio < 0.6）会跳过部分边缘点

### 估算浮力点数量：
- 船头和船尾区域（zRatio > 0.6）：约4个位置 × 3个点 = 12个点
- 船中区域（zRatio < 0.6）：约6个位置，但会跳过部分边缘点
  - 中心线点（x=0）：6个位置全部保留 = 6个点
  - 边缘点（x≠0）：约3个位置保留 = 6个点
- 强制添加：船头中心、船尾中心 = 2个点
- **总计约：12 + 6 + 6 + 2 = 26个点**（去重后可能更少）

### 问题：
对于138米长、85米宽的大型双体船，**26个浮力点太少**！

## 3. 浮力计算推导

### 当前参数：
- `buoyancyCoeff = 50,000`
- `maxBuoyancy = body.mass × 9.82 × 3.0 = 1,090,020,000 N`
- `effectivePointCount = buoyancyPoints.length × 0.3`
- `maxForcePerPoint = maxBuoyancy / effectivePointCount`

### 假设有26个浮力点：
- `effectivePointCount = 26 × 0.3 = 7.8 ≈ 8`
- `maxForcePerPoint = 1,090,020,000 / 8 = 136,252,500 N/点`

### 实际浮力计算：
对于每个在水下的浮力点：
```
calculatedForce = depth × buoyancyCoeff × density × positionWeight
                = depth × 50,000 × 1.0 × positionWeight
```

### 问题分析：

**情况1：船体完全在水面以上**
- 所有浮力点的 `depth ≤ 0`
- 没有浮力产生
- 船体受重力作用下沉

**情况2：船体部分入水（假设吃水1米）**
- 假设有10个点在水下，平均深度0.5米
- 每个点的浮力 = 0.5 × 50,000 = 25,000 N
- 总浮力 = 10 × 25,000 = 250,000 N
- **远小于重力 363,340,000 N！**

**情况3：船体达到11米吃水**
- 假设有20个点在水下，平均深度5.5米
- 每个点的浮力 = 5.5 × 50,000 = 275,000 N
- 但受限于 `maxForcePerPoint = 136,252,500 N`
- 实际每个点浮力 = min(275,000, 136,252,500) = 275,000 N
- 总浮力 = 20 × 275,000 = 5,500,000 N
- **仍然远小于重力 363,340,000 N！**

## 4. 根本问题

1. **浮力点数量太少**：26个点对于138米长的船远远不够
2. **浮力系数不够大**：即使50,000，在深度较小时产生的浮力仍然不足
3. **浮力点分布可能不合理**：船体下沉时，可能只有少数点在水中

## 5. 解决方案

### 方案1：大幅增加浮力点数量
- 将 `lengthSegments` 增加到20+，确保有足够多的浮力点
- 增加横向采样点（从3个增加到5个或更多）

### 方案2：大幅增加浮力系数
- 将 `buoyancyCoeff` 增加到 200,000 或更高
- 确保在浅水时也能产生足够浮力

### 方案3：调整浮力计算逻辑
- 降低 `effectivePointCount` 比例（从0.3降到0.2）
- 或者移除每个点的浮力上限限制

### 方案4：组合方案
- 增加浮力点数量 + 增加浮力系数 + 调整上限计算
