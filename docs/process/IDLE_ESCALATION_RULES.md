# Idle Escalation Rules（无产出升级规则）

**创建时间**: 2026-03-15 11:02

## 1. 目的
防止出现长时间无硬产出却仍维持“项目正常推进”叙事的情况。

## 2. 时间阈值
### 30 分钟
- 允许处于实现/排障/重构中
- 但下一次汇报必须给出至少一项硬证据

### 60 分钟
- 若仍无硬证据，状态标记为：**轻微失速**
- 必须解释当前卡点

### 120 分钟
- 若仍无硬证据，状态标记为：**明显失速**
- 必须缩减范围为 P0
- 不允许再使用“按计划推进中”表述

## 3. 硬证据定义
- 文件新增或修改
- 测试输出
- API 返回
- 页面可见变化
- 运行日志

## 4. 不被接受的替代物
- 纯计划更新
- 空泛状态汇报
- 仅描述“在思考/在整理/在查看”
- 只说“进行中”但无证据

## 5. 升级动作
- 失速 60 分钟：下一时间片只做一个 P0 交付
- 失速 120 分钟：暂停扩展目标，优先恢复最小闭环
- 失速 180 分钟以上：必须提交复盘并重做计划
