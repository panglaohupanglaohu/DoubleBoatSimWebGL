## 核心知识点

### 1. 单元测试原则
- **独立性**：每个测试用例应独立运行，不依赖其他测试
- **可重复性**：测试结果应始终一致，不受环境影响
- **自验证**：测试应自动判断通过/失败，无需人工检查
- **及时性**：测试应与代码同时编写（TDD 理念）

### 2. 测试覆盖类型
- **语句覆盖**：确保每行代码至少执行一次
- **分支覆盖**：确保每个条件分支都测试到
- **边界值测试**：测试输入范围的边界情况
- **异常测试**：测试错误处理和边界条件

### 3. 测试命名规范
- 使用描述性名称说明测试目的
- 格式：`test_[功能]_[条件]_[预期结果]`
- 中文注释说明测试场景

### 4. 断言最佳实践
- 每个测试聚焦一个断言点
- 使用有意义的错误消息
- 避免过度复杂的断言逻辑

## 可应用到项目的技术

- **unittest 框架** - Python 标准库，无需额外依赖
- **setUp/tearDown** - 测试前后准备和清理
- **测试套件** - 批量运行多个测试文件
- **覆盖率报告** - 使用 coverage.py 生成报告
