💡 性能优化建议
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. 关键词提取优化:
   - 预编译正则表达式，避免重复编译
   - 使用集合 (set) 进行停用词查找，O(1) 复杂度
   - 考虑使用更高效的字符串分割方法

2. 关键词匹配优化:
   - 使用 Aho-Corasick 算法进行多模式匹配
   - 对文档建立倒排索引，加速关键词查找
   - 使用缓存存储频繁查询的结果

3. 整体架构优化:
   - 考虑异步处理，提高并发能力
   - 实现结果缓存，减少重复计算
   - 使用更高效的字符串拼接方法 (join vs +)

4. 日志系统优化:
   - 使用异步日志，避免阻塞主线程
   - 实现日志缓冲，减少 I/O 操作
   - 根据日志级别动态调整详细程度
